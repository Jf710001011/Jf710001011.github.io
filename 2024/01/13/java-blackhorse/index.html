  7. str.charAt(i) vs str[i]

    1. Immutability: str.charAt(i) doesn't modify the original string, while str[i] can potentially modify it if used for assignment.
    2. Readability: str.charAt(i) is often considered more readable, especially for those new to Java, as it clearly conveys the intent of accessing a character within a string.
    3. Potential for Errors: str[i] can lead to NullPointerException if the string is null, while str.charAt(i) throws a more specific IndexOutOfBoundsException for invalid indices.

  10. What is StringBuilder?

  11. A mutable string class, meaning you can modify its contents after creation.

  12. Designed for efficient string manipulation, especially when performing frequent modifications.

  13. It's a better choice than String for repeated string concatenations as it avoids creating multiple String objects in memory.

  14. Common Use Cases:

    1. Dynamic string building, like assembling HTML content or generating log messages.
    2. Reading and processing text files or user input line by line.
    3. Looping and appending to strings within loops.

  15. Be mindful of thread safety if using StringBuilder in multi-threaded environments.

  19. String splicing principle

    package String_Pratice;

public class StringSplicingPrinciple {

    public static void main(String[] args) {

        // If there are no variables involved, strings are added directly. After
        // compilation, the result is the spliced ​​result.
        // Will reuse strings in the string pool (in the constant pool).
        String s1 = "Hello" + "World";
        String s2 = "HelloWorld";
        System.out.println(s1 == s2); // true
        System.out.println("--------");

        // If there are variables involved
        // Each line of concatenated code creates a new string in memory.
        String s3 = "Hello";
        String s4 = s3 + "World";
        System.out.println(s2 == s4); // false
        System.out.println("--------");

        // All content that needs to be spliced will be put into StringBuilder,
        // which will not create a lot of useless space and save memory.
        StringBuilder sb = new StringBuilder();
        sb.append("Hello");
        sb.append("World");
        String s5 = sb.toString();
        System.out.println(s5 == s2); // false
        System.out.println("--------");
    }

}

  20. StringBuilder Expansion

    package String_Pratice;

public class StringBuilderExpansion {
    public static void main(String[] args) {
        // StringBuilder expansion

        // If the length of the added content is less than 16, it will be stored
        // directly.

        StringBuilder sb = new StringBuilder();

        // By default, a byte array of length 16 is created
        System.out.println(sb.capacity()); // 16

        // If the added content is greater than 16, the capacity will be expanded
        // (original capacity * 2 + 2)
        sb.append("HelloWorldHello");
        System.out.println("capacity: " + sb.capacity()); // 16
        System.out.println("length: " + sb.length()); // 15

        sb.append("HelloWorldHello");
        System.out.println("capacity: " + sb.capacity()); // 34
        System.out.println("length: " + sb.length()); // 15+15

        // If it is not enough after expansion, the actual length shall prevail.
        sb.append("HelloWorldHelloHelloWorldHelloorldHellolo");
        System.out.println("capacity: " + sb.capacity()); // 30+41>(34*2+2)
        System.out.println("length: " + sb.length()); // 30+41

        System.out.println("--------");
    }
}
