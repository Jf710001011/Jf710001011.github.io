padding oracle 原理深度解析 & CBC 字节翻转攻击原理解析

  1. Padding的含义是“填充”，在解密时，如果算法发现解密后得到的结果，它的填充方式不符合规则，那么表示输入数据有问题，对于解密的类库来说，往往便会抛出一个异常，提示Padding不正确。Oracle在这里便是“提示”的意思

  2. padding oracle漏洞的就是CBC加密模式

  3. AES是对称加密，对称加密呢有两大类加密模式，即分组加密和流加密，AES分组加密有五种加密模式:

    1. 电码本模式（Electronic Codebook Book (ECB)）；
    2. 密码分组链接模式（Cipher Block Chaining (CBC)）；
    3. 计算器模式（Counter (CTR)）；
    4. 密码反馈模式（Cipher FeedBack (CFB)）；
    5. 输出反馈模式（Output FeedBack (OFB)）

  4. AES是加密算法，加密算法是通过接收方和发送方协商产生密钥，结合一系列的各种位运算之后得出的结果。而加密模式是在加密算法的基础上，把加密的方式变得更加复杂一点

  5. CBC模式是一个链式结构

    1. 在通过AES加密之前我们需要先将明文块0，也就是第一块明文，和我们的初始向量IV做异或操作，这个初始向量IV是随机的，而且长度是和我们的每一块明文块等长，因为要按位进行异或。这样无疑就在加密之前就已经先行打乱的我们的明文，与初始化向量异或后的明文，我们暂且称它为中间值，我们此时再对这个中间值进行AES加密，这样第一块明文的加密就完成了。
    2. 我们将第一块明文加密后的密文，作为第二个明文加密时的初始化向量，而这个就是这个链式结构的连接点，后续的步骤就是不断重复加密第一块明文时所做的操作，直至最后一块明文加密完成

    但是CBC这个模式在设计上存在缺陷 ，而这个缺陷就导致了著名的padding oracle攻击

  6. 用AES和DES两个加密算法来举例子 AES的分组长度为每块16字节，DES呢则是每块8字节，那么怎么能保证我们的明文长度是16或者8的整数倍呢？

    当然没办法保证，所以我们就要采取措施强制让明文为16或者8的整数倍，最直接的方法自然就是直接填充，不够就补到它够为止。这也就是所谓的padding 填充。

  7. 剩余的几位当然不能随便填充，而要填充一些有价值的数值。

  8. 假如说此时我们按8字节为一个明文分组，分到最后发现最后一组缺了一个字节，程序不会填一些随机数，亦或者将不够的位数全填零。CBC模式最后的填充方法，就是缺了一位就填一个0x01,缺了两位就填两个0x02,缺了三位就填三个0x03,以此往后类推缺n个就填n个0x0n。哪怕当明文正好时分组的整数倍时，也会填充8个0x08，即使是整数倍也要填充。这样就导致了无论我们明文的长度是多少，我们CBC模式加密是都会在明文的最后进行填充，以确保分段的长度是8的整数倍。

  9. 而padding oracle攻击，同样是通过服务端返回的信息的差异而产生的

  10. 目标：不知道密钥的情况下得到这段密文的明文，即我们是不知道中间值的

  11. 刚才的已知条件中，我提到了初始化向量必须已知才能进行攻击，那么这个初始化向量在哪呢？

    一般是在密文的头部

  15. 将初始化向量全部制为零,异或的结果是3D，此时我们是不可能知道这个异或出来的结果是3D的，但是此时服务端会报一个错，那就是填充位错误，为什么会报这个错，因为之前说了，填充文在8个字节为分组的情况下，最多只可能填到8个0x08，所以怎么可能会有0x3D呢？

  16. 如果此时异或出来的结果为 “39732322076a2601”也就是异或出来的结果最后一位为0x01时就不会报填充位错误了。但是后续还会在爆一个错误，那就是业务判断你这个解密出来的明文数据也就是“39732322076a2601”不正确，因为我们初始化向量都制为零的，所以这个明文当然是错误的，不过这都不重要。

    用于对密文进行反向推理攻击，主要参数解析:

    • padbuster: 表示使用padbuster工具进行padding oracle attack

    • -encoding 0: 指定密文的编码方式为0

    攻击思路是:

    1. 发送密文,并观察响应差异判断填充校验结果

    2. 修改密文的最后一个字节(16位置),再次发送,直到填充正确

    3. 重复此流程,可以逐字节反推出正确的明文

    4. 最终可以完全恢复明文信息

    这样通过分析密文和填充校验响应,不需要知道密钥就可以破译密文,达到攻击目的。

  1. 对main.py代码审计

    1. 控制 fns=["xx || ls"]： 命令注入

      1. rep += ...: 这是一个字符串拼接操作，rep是一个字符串变量，通过+=操作符将后面的内容追加到该变量中。

      2. '<i>Space used: ': 这是一个HTML标签<i>，用于表示斜体文本，后面跟着字符串 "Space used: "。

      3. subprocess.check_output(...): 这是Python的subprocess模块，用于运行系统命令并获取其输出。在这里，它执行了一个du命令来计算指定目录的磁盘空间使用情况。

      4. 'du -ch %s || exit 0' % ' '.join('files/' + fn for fn in fns): 这是一个命令字符串，%s是一个占位符，将由后面的' '.join('files/' + fn for fn in fns)所替换。'files/' + fn for fn in fns是一个列表生成式，将在每个文件名fn前添加"files/"，形成一个新的字符串列表，这些文件列表将传递给du命令来计算它们的磁盘空间使用情况。

      5. shell=True: 这是告诉subprocess.check_output()在一个shell中执行命令，这样可以支持管道和其他shell特性。

      6. stderr=subprocess.STDOUT: 这是将标准错误输出与标准输出合并，这样错误信息将显示在输出结果中，而不会产生异常。

      7. .strip().rsplit('n', 1)[-1]: 这是对subprocess.check_output()返回的结果进行处理，首先通过.strip()方法去除输出结果中的空白字符，然后通过.rsplit('n', 1)[-1]将输出结果按照换行符分割，并取最后一个部分（最后一行）作为结果。

      8. '</i>': 这是一个HTML结束标签，用于结束斜体文本的表示。 综上所述，该代码段的目的是获取指定目录下文件的磁盘空间使用情况，并将结果以斜体文本的形式添加到rep字符串变量中。请注意，代码执行的前提是系统中有可用的du命令，并且fns变量包含了要统计空间使用情况的文件列表。

    2. 控制fns

      解析：

      1. cur.execute(...): 这是在Python中使用数据库的典型方式，cur是数据库的游标对象，它执行SQL查询。这里执行了一个SELECT查询，从名为"photos"的数据库表中选择指定"parent"值的记录，最多选择3条记录。"parent"值是通过占位符 %s 指定的，它将由后面的(id,)所替换。

      2. fns = []: 这是一个空列表，用于存储从数据库中检索到的文件名。

      3. for pid, ptitle, pfn in cur.fetchall():: 这是对数据库查询结果进行迭代的循环，cur.fetchall()返回所有满足查询条件的结果行。在每次迭代中，pid表示照片的ID，ptitle表示照片的标题，pfn表示照片的文件名。

      4. rep += ...: 这是字符串拼接操作，将HTML格式的照片信息添加到rep字符串变量中。

      5. '<div><img src="fetch?id=%i" width="266" height="150"><br>%s</div>' % (pid, sanitize(ptitle)): 这是一个HTML标签<div>，用于创建一个包含照片和标题的块。<img>标签用于显示照片，它的src属性是一个URL，指向一个用于获取照片内容的"fetch"服务，%i是一个占位符，将由后面的pid所替换，用于指定照片的ID。width和height属性指定照片的显示宽度和高度。<br>标签用于插入一个换行。%s是一个占位符，将由后面的sanitize(ptitle)所替换，用于显示经过处理的照片标题。

      6. fns.append(pfn): 将每个照片的文件名 pfn 添加到 fns 列表中，以便后续使用。 综上所述，该代码段的目的是从数据库中选择指定"parent"值的照片记录，并将它们以HTML格式添加到rep字符串变量中。同时，也将照片的文件名添加到fns列表中，以供后续使用。

    3. fns的项来自表photos中filename

      • python def fetch(): cur = getDb().cursor() - 定义了一个名为 fetch 的函数。
      • 调用 getDb() 函数来获取数据库连接对象。
      • 使用 cursor() 方法获取一个游标对象 cur，用于执行SQL查询。
      • python #这里存在注入。我们可以控制request.args['id']达到控制sql过程 if cur.execute('SELECT filename FROM photos WHERE id=%s' % request.args['id']) == 0: abort(404) - 这是一个注释，指出代码存在SQL注入漏洞。因为在查询中使用了未经处理的用户输入 request.args['id']。
      • cur.execute() 执行SQL查询，使用 %s 占位符来表示需要传递的参数，但是这里直接将用户输入 request.args['id'] 直接插入查询中，存在SQL注入的风险。
