  2. generic paging entity

    Mybatis Plus provides a convenient and flexible approach to pagination, enabling developers to efficiently retrieve paginated data from databases. The "universal paging entity" (通用分页实体) plays a central role in this process by encapsulating pagination parameters and facilitating their consistent usage across different mapper methods and service calls.

    Key Features of the Universal Paging Entity:

    1. Simplified Pagination Logic: By encapsulating pagination parameters in a dedicated entity, developers can manage and pass these parameters consistently, reducing the complexity of pagination logic.
    2. Enhanced Code Readability: Using a separate entity for pagination parameters improves code readability and maintainability, making it easier to understand and modify paging behavior.
    3. Promotes Code Reuse: The universal paging entity can be reused across various parts of the application, minimizing code duplication and promoting consistent pagination handling.

    Structure of the Universal Paging Entity:

    The specific structure of the universal paging entity may vary depending on the application's requirements and preferences. However, it typically includes the following fields:

    1. page: Represents the current page number.
    2. size: Represents the page size, i.e., the number of items per page.
    3. orderByColumn: Specifies the column name for sorting.
    4. isAsc: Indicates whether the sorting order is ascending (true) or descending (false).

    Example generic paging entity:

    Java

    public class PageInfo<T> {

    private Long page;
    private Long size;
    private String orderByColumn;
    private Boolean isAsc;

    // Getters and setters for all fields
}

    Using the generic paging entity:

    To utilize the universal paging entity, follow these steps:

    1. Create the Paging Entity: Create a class that represents the paging entity, including the necessary fields for page number, page size, and sorting.
    2. Pass Paging Entity to Mapper: When invoking mapper methods that require pagination, pass an instance of the paging entity as an argument.
    3. Process Paging Parameters: Mybatis Plus will automatically extract the pagination parameters from the paging entity and apply them to the corresponding SQL query.

    Example Usage:

    Java

    // Create paging entity instance
PageInfo<User> pageInfo = new PageInfo<>();
pageInfo.setPage(1L);
pageInfo.setSize(10L);
pageInfo.setOrderByColumn("name");
pageInfo.setIsAsc(true);

// Retrieve paginated user list
List<User> users = userService.selectPage(pageInfo);

// Access pagination information from pageInfo object
System.out.println("Total pages: " + pageInfo.getPages());
System.out.println("Total records: " + pageInfo.getTotal());

    By employing the universal paging entity, developers can streamline pagination logic, enhance code readability, and promote code reuse in Mybatis Plus applications.

Functions

  1. Routing: Directs client requests to the appropriate microservice based on the request URL and other criteria.
  2. Security: Handles authentication and authorization.
  3. Load Balancing: Distributes requests among instances of a microservice.
  4. Rate Limiting: Controls the rate of incoming requests to prevent overloading services.
  5. Logging and Monitoring: Captures and logs request and response details for monitoring and debugging.
  6. Transformation: Modifies requests and responses, such as URL rewriting and header manipulation.

Workflow

  1. Microservice Registration:
    • Microservice A starts and registers itself with Nacos.
    • Microservice B starts and registers itself with Nacos.
  2. Client Request to API Gateway:
    • A client sends a request to the API Gateway.
  3. Service Discovery:
    • The API Gateway queries Nacos to find the instances of the required microservice (e.g., Service A).
  4. Load Balancing:
    • The API Gateway selects an instance of Service A using a load balancing strategy.
  5. Routing:
    • The API Gateway forwards the client request to the selected instance of Service A.
  6. Service-to-Service Communication:
    • If Service A needs to communicate with Service B, it queries Nacos to discover instances of Service B.
    • Service A then sends a request to an instance of Service B.

    package com.hmall.gateway.filters;

import java.util.List;

import org.springframework.cloud.gateway.filter.GatewayFilterChain;
import org.springframework.cloud.gateway.filter.GlobalFilter;
import org.springframework.core.Ordered;
import org.springframework.http.HttpStatus;
import org.springframework.http.server.reactive.ServerHttpRequest;
import org.springframework.http.server.reactive.ServerHttpResponse;
import org.springframework.stereotype.Component;
import org.springframework.web.server.ServerWebExchange;

import com.hmall.common.exception.UnauthorizedException;
import com.hmall.gateway.config.AuthProperties;
import com.hmall.gateway.utils.JwtTool;

import cn.hutool.core.text.AntPathMatcher;
import lombok.RequiredArgsConstructor;
import reactor.core.publisher.Mono;

@Component
@RequiredArgsConstructor
public class AuthGlobalFilter implements GlobalFilter, Ordered {

    private final AuthProperties ap;
    private final JwtTool jt;
    private final AntPathMatcher apm = new AntPathMatcher();

    @Override
    public int getOrder() {
        return 0;
    }

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        ServerHttpRequest request = exchange.getRequest();
        if (isExclude(request.getPath().toString())) {
            return chain.filter(exchange);
        }

        String token = null;
        List<String> headers = request.getHeaders().get("authorization");
        if (headers != null && !headers.isEmpty()) {
            token = headers.get(0);
        }

        Long userId = null;
        try {
            userId = jt.parseToken(token);
        } catch (UnauthorizedException e) {
            ServerHttpResponse response = exchange.getResponse();
            response.setStatusCode(HttpStatus.UNAUTHORIZED);
            return response.setComplete();
        }

        System.out.println("userId=" + userId);
        return chain.filter(exchange);
    }

    private boolean isExclude(String path) {
        for (String pathPattern : ap.getExcludePaths()) {
            if (apm.match(pathPattern, path)) {
                return true;
            }
        }
        return false;
    }

}

    package com.hmall.gateway.route;

import java.util.HashSet;
import java.util.List;
import java.util.Set;
import java.util.concurrent.Executor;

import javax.annotation.PostConstruct;

import org.springframework.cloud.gateway.route.RouteDefinition;
import org.springframework.cloud.gateway.route.RouteDefinitionWriter;
import org.springframework.stereotype.Component;

import com.alibaba.cloud.nacos.NacosConfigManager;
import com.alibaba.nacos.api.config.listener.Listener;
import com.alibaba.nacos.api.exception.NacosException;
import com.hmall.common.utils.CollUtils;

import cn.hutool.json.JSONUtil;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import reactor.core.publisher.Mono;

@Slf4j
@Component
@RequiredArgsConstructor
public class DynamicRouteLoader {
    private final RouteDefinitionWriter rdw;
    private final NacosConfigManager ncm;

    private final String dataId = "gateway-routes.json";
    private String group = "DEFAULT_GROUP";

    private final Set<String> routeIds = new HashSet<>();

    @PostConstruct
    public void initRouteConfigListener() throws NacosException {
        // Register the listener and pull the configuration for the first time
        String configInfo = ncm.getConfigService().getConfigAndSignListener(dataId, group, 5000, new Listener() {

            @Override
            public Executor getExecutor() {
                return null;
            }

            @Override
            public void receiveConfigInfo(String configInfo) {
                updateConfigInfo(configInfo);
            }

        });
        updateConfigInfo(configInfo);
    }

    private void updateConfigInfo(String configInfo) {
        log.debug("Monitor routing configuration changes,{}", configInfo);

        // 1. Deserialization
        List<RouteDefinition> routeDefinitions = JSONUtil.toList(configInfo, RouteDefinition.class);

        // 2. Clear old routes before updating
        // 2.1 Clear old routes
        for (String routeId : routeIds) {
            rdw.delete(Mono.just(routeId)).subscribe();
        }
        routeIds.clear();
        // 2.2 Determine whether there are new routes to be updated
        if (CollUtils.isEmpty(routeDefinitions)) {
            // No new routing configuration, end
            return;
        }

        // 3. update routes
        routeDefinitions.forEach(routeDefinition -> {
            // 3.1 update routes
            rdw.save(Mono.just(routeDefinition)).subscribe();
            // 3.2 Record the route ID for future deletion
            routeIds.add(routeDefinition.getId());
        });
    }
}
